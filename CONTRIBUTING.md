## âš™ï¸ Bagaimana Cara Berkontribusi?

Ada banyak cara untuk membantu, tidak hanya melalui kode\!

### 1\. Pelaporan *Bug* (Masalah) ðŸ›

Jika kamu menemukan kesalahan (bug) pada *workflow* otomatis, *template* HTML, atau tautan yang rusak:

  * Gunakan tab **[Issues di GitHub](https://github.com/frijal/LayarKosong/issues)**.
  * Berikan deskripsi yang jelas dan langkah-langkah untuk mereproduksi *bug* tersebut (jika memungkinkan).
  * Gunakan label (`bug`) yang sesuai.

### 2\. Diskusi & Ide Baru ðŸ’¡

Punya ide keren untuk *workflow* baru, fitur SEO, atau ingin membahas teknologi?

  * Gunakan tab **[Discussions di GitHub](https://github.com/frijal/LayarKosong/discussions)**.
  * Ini adalah tempat terbaik untuk *brainstorming* sebelum menulis kode.

### 3\. Kontribusi Kode (Pull Request) ðŸ’»

Jika kamu ingin menyumbang kode, seperti perbaikan *bug*, pembaruan *workflow* YAML, atau peningkatan *style* CSS:

#### A. Persiapan

1.  **Fork** Repository ini ke akun GitHub-mu.
2.  **Clone** *repo* yang sudah di-*fork* ke komputer lokalmu.
3.  Buat *branch* baru untuk perubahanmu (misalnya: `feat/perbaikan-css` atau `fix/workflow-bug`).

#### B. Aturan Penulisan Kode

  * **HTML/Markdown:** Prioritaskan kesederhanaan. Pastikan kode HTML-mu valid dan bersih.
  * **CSS:** Karena ini proyek *static site* super ringan, usahakan menggunakan CSS yang efisien atau *inline/scoped* jika diperlukan.
  * **Workflow (.yml):** Pastikan skrip YAML yang kamu buat/ubah berfungsi dan aman.

#### C. Mengirimkan Pull Request (PR)

  * Pastikan *commit* kamu memiliki pesan yang jelas (misalnya: `fix: Perbaiki link favicon yang rusak` atau `feat: Tambahkan action ping sitemap`).
  * Buka **Pull Request** ke *branch* `main` dari *repository* utama.
  * **Deskripsikan** PR-mu dengan detail: Apa yang kamu ubah? Kenapa kamu mengubahnya? Apa hasilnya?

## ðŸ“ Kontribusi Konten (Artikel)

Karena **Layar Kosong** fokus pada publikasi konten HTML, kami menyambut baik kontribusi artikel/tulisan\!

1.  Buat file HTML baru (misalnya `artikel-judul-mu.html`).
2.  Letakkan file tersebut di folder **`artikelx/`**.
3.  Kirimkan **Pull Request**.

> **Penting Mengenai Lisensi Konten:** Karena proyek ini menggunakan **UNLICENSE** (Public Domain), dengan mengirimkan artikel ke folder `artikelx/`, kamu secara otomatis melepaskan hak cipta atas tulisanmu tersebut ke Domain Publik. Pastikan kamu nyaman dengan hal ini\!
